## How to install the CA certificate in certificate manager in windows?

To intercept HTTPS network traffic, you need to install the Requestly CA (Certificate Authority) certificate to your Windows certificate manager. There are two methods: automatic and manual.

### Method 1: Automatic Installation (Recommended)

<Steps>
  <Step title="Open Requestly Desktop App">
    Launch the Requestly Desktop App on your Windows system
  </Step>
  
  <Step title="Enable System-wide Proxy">
    Click on **"Enable Requestly System-wide"**
    <img src="/images/install-certificate-windows/requestly-desktop-app-windows.webp" alt="Requestly Desktop App Windows" />
  </Step>
  
  <Step title="Install & Trust Certificate">
    - Click on **"Install & Trust"** button
    - A popup will automatically appear to install the certificate
    - Follow the prompts to complete the installation
    <img src="/images/install-certificate-windows/install-and-trust-certs.webp" alt="Install and Trust Certificate" />
  </Step>
</Steps>

### Method 2: Manual Installation

If the automatic method doesn't work, follow these manual steps:

<Steps>
  <Step title="Download the CA certificate">
    - In Requestly Desktop App, go to the System Proxy popup
    - Click on **Manual** tab
    - Click on **Save Certificate** button
    - Save the CA certificate to any location on your system (e.g., Desktop or Downloads folder)
    <img src="/images/install-certificate-windows/manual-certificate-installation.webp" alt="Manual Certificate Installation" />
  </Step>
  
  <Step title="Import the CA certificate in Certificate Manager">
    - Press `WIN + R`, type `certmgr.msc` and press Enter
    - Navigate to: **Trusted Root Certification Authorities → Certificates**
    - Right-click on **Certificates** → **All Tasks → Import**
    - Browse and select the CA certificate file you downloaded in step 1
    - Click **Finish**
    <img src="/images/install-certificate-windows/install-win-cert.gif" alt="Install Certificate in Windows Certificate Manager" />
  </Step>
  
  <Step title="Verify the installation">
    - You should see an "Import was successful" message
    - To verify if HTTPS traffic is being intercepted, connect to the proxy
    - Check that secure (HTTPS) traffic is now being logged in Requestly
    <img src="/images/install-certificate-windows/intercepted-traffic.webp" alt="Intercepted HTTPS Traffic" />
  </Step>
</Steps>

### Troubleshooting

If you're still unable to intercept HTTPS traffic after following these steps, please ensure:
- The CA certificate was imported to the correct location (Trusted Root Certification Authorities)
- The proxy is properly configured and running
- Your browser or application trusts system certificates
- Try restarting your browser after CA certificate installation
